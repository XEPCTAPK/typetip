## 🏗️ 1. ТЕХНИЧЕСКАЯ ФИЛОСОФИЯ И АРХИТЕКТУРА ЯДРА

Проект `typetip` представляет собой изолированную, высокопроизводительную среду выполнения текстовых консольных интерфейсов (TUI) непосредственно внутри сессии расширения VS Code. Вместо использования классического статического связывания (Tight Coupling), архитектура построена на принципах **Zero-Static-Coupling** и динамического разрешения зависимостей на этапе выполнения (Runtime Reflection).

### Ключевые паттерны системы:
* **Lazy Module Loading:** Загрузка исполняемых экранов происходит асинхронно через перехват кэша модулей Node.js. Это исключает раздувание оперативной памяти долгоживущей сессии редактора.
* **Low-Level PTY Lock:** Аппаратный контроллер фокуса удерживает низкоуровневый поток ввода псевдотерминала в нижней панели, изолируя управляющие последовательности `W/S/A/D` от глобальных горячих клавиш IDE.
* **Bi-directional Canvas Render:** Двунаправленный конвейер отрисовки кадра совмещает базовый графический слой активного экрана с динамическими оверлеями системной консоли.

## 📁 2. ДЕТАЛЬНАЯ СПЕЦИФИКАЦИЯ МОДУЛЕЙ И СТРУКТУРЫ

Согласно архитектурной карте проекта, исходный код разделен на изолированные домены ответственности:

### ⚙️ Подсистема Ядра (`/src/core/` & `/src/engine/`)
* `router.ts` — **Динамический маршрутизатор ядра.** Управляет жизненным циклом экранов, автоматически зачищает буфер (`\x1b[0m\x1b[2J\x1b[3J\x1b[H`) и выполняет горячую перезагрузку классов без перезапуска VS Code.
* `consol.ts` — **Инженерная консоль разработчика (DevConsole).** Сквозной управляющий слой, осуществляющий перехват ввода по клавише `` ` `` (тильда) и инжекцию отладочных данных поверх графического фрейма.
* `terminal.ts` & `graphics.ts` — Низкоуровневый абстрактный слой над псевдотерминалом VS Code, отвечающий за ANSI-экранирование, позиционирование каретки и отрисовку матриц.
* `netScanner.ts` & `netMonitor.ts` — Модули фонового сканирования и сетевого мониторинга топологии системы.
* `reservation.ts` & `storage.ts` — Компоненты персистентного сохранения состояний, резервирования данных и долгоживущих сессий пользователя.
* `layouts.ts` — Менеджер конфигурации физических раскладок для тренировки слепой печати.

### 🎮 Модульные Экраны (`/src/screens/`)
* `hubmenu.ts` — Главный координационный центр. Автоматически сканирует доступные исполняемые файлы и формирует интерактивную сетку выбора.
* `intro.ts` & `intro2.ts` — Базовый текстовый промпт инициализации и альтернативный графический экран (Глитч-Матрица).
* `keyboard.ts` & `keyboard2.ts` — Классический тренажер кодинга №1 и модуль кастомных раскладок №2.
* `dash.ts` — Режим блиц-спринта по конструкциям кода и проверке PEP8-стандартов.
* `rain.ts` — Интерактивный дебаг-режим: уничтожение падающих символов на скорость.
* `versus.ts` — Экран кибер-дуэлей против ИИ-ботов (в режиме активной разработки).
* `crew.ts` — Интерфейс управления командным синдикатом разработчиков.

### 🖼️ Статические Ассеты (`/src/screens/assets/`)
* `BIG_ANSI_ART.txt` — Мастер-файл крупногабаритной ANSI-графики, рендерящийся на лету в разметку терминала при вызове модуля `ASSET_PREVIEW`.

## 🛠️ 3. ИНСТРУКЦИЯ ПО РАЗВЕРТЫВАНИЮ И СБОРКЕ

Для локальной отладки и компиляции ядра выполните следующие шаги:

```bash
# 1. Клонирование репозитория в локальную директорию
git clone [https://github.com/XEPCTAPK/typetip.git](https://github.com/XEPCTAPK/typetip.git)

# 2. Установка зависимостей сборщика и типов VS Code
npm install

# 3. Компиляция TypeScript исходников в исполняемый JS (директория /out/)
npm run compile

# 4. После завершения компиляции нажмите F5 в VS Code для запуска расширения в режиме хост-отладки (Extension Development Host)
